# Self-hosted concept: TASKPLAN PRO Operation Map

This example documents the already implemented and released behavior of
`taskplan-pro-operation-map`. It is not a fictional product proposal and does
not extend the package scope. The owner authorized this public self-hosted
example on 2026-07-21.

## C01. Vendor-neutral agent skill and TASK-PLAN PRO module

TASKPLAN PRO Operation Map is both a vendor-neutral skill for Codex, Claude
Code, and other compatible coding-agent environments, and a standalone module
of the broader TASK-PLAN PRO framework. The skill guides an agent through
concept decomposition and readiness; the packaged CLI provides deterministic
validation and projection commands. This repository does not contain or claim
to be the complete TASK-PLAN PRO product.

## C02. Accepted concept to validated operation graph

Given an accepted product concept, the module represents the complete product
journey as one canonical JSON graph. Stable blocks and nodes describe inputs,
outputs, success criteria, failure criteria, recovery routes, ownership,
evidence, and traceability to concept sections. Readiness fails when required
meaning, traceability, gate routes, source identity, or implementation evidence
is missing.

## C03. Deterministic review projections

Only a readiness-admissible graph may produce the derived Markdown map,
presentation data, locale catalog, build manifest, and self-contained HTML
review workspace. The HTML supports overview and block drill-down, stable-node
comments, local autosave, JSON import/export, and saved reviewed HTML without a
backend or telemetry.

## C04. Evidence honesty and scope boundary

Canonical JSON remains the source of truth; Markdown and HTML are projections.
The module does not invent missing product meaning, execute an entire project,
replace product-owner acceptance, or claim the capabilities of the complete
TASK-PLAN PRO framework. Implementation claims must resolve to repository
files and passing tests.
